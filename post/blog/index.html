Hugo+Github个人主页的搭建过程
2020-07-20 15:53

前言

很早以前我就有搭建一个个人博客的想法，用来记录一下自己的学习过程和学习成果，恰巧最近有空闲的时间，于是我开始了这次实践。对于建站我属于完全零基础，所以一切都要从头学起。

自己写or用工具搭建？

这个是我第一个纠结的地方，说纠结其实也不然，因为我根本就不会自己写（菜鸡流泪），所以如果要自己写的话，要花不少的时间成本去学习html、css、jsp等等，所以我决定利用工具和模板，搭建一个自己的个人主页，我觉得可以在搭建完再反过来利用现成的代码来学习上述的那些建站需要的技能，作为学习的实例。

为什么我选择了Hugo

在参观别人的blog时，我偶然看到了一个用hugo搭建的blog，这里可以放个链接：nanshu.wang，整个页面，无论是排版还是主题风格都和我理想中的blog相差无几，在对于建站工具没有更深了解的情况下，我直接选择了hugo，当然在选择后我了解到了Hugo的一些优点，这也使我坚定了用Hugo建站的想法。

  • Hugo 使用的GO语言是比较新的语言；
  • Hugo 是最新的生成器，功能方面也肯定集成了之前其他生成器的优点；
  • 不用安装，windows下只需要一个exe程序，就可以生成静态文件了，建站过程方便。

Hugo的安装和配置

因为关于Hugo的安装教程在网上可以找到很多文章，而且并没有什么操作难度，也几乎不会碰到什么问题，只是不要忘记配置环境变量，所以这一部分我只简略说说。 首先进行下载安装,访问Hugo，选择相应的版本下载并解压。

接着将hugo.exe所在目录添加到环境变量path后面

然后打开win+rcmd，输入

hugo version

如果返回相应的版本信息则说明安装配置成功。

除此以外还需要安装 Git（下载链接）

用来将生成好的网页推送至github服务器，相应的安装教程网上都可以找到，此处不再赘述。

Hugo静态网页生成

Hugo自称The world’s fastest framework for building websites，在生成网页和配置网页的过程中，我感觉Hugo的语法的确很简单，没有什么繁琐的需要记忆的代码，网页的生成速度也很快，我逐渐体会到了Hugo自称的fastest。在物理和心理两个角度，给予使用者更多的便捷，这可能就是Hugo火起来的原因吧。废话不多说，进入正题。

1.网页生成、初识目录：

  • 方法一：运行cmd.exe，进入hugo所在目录，输入下方指令，即可在hugo目录下生成一个blog文件夹，即静态网页文件。
hugo new site blog
  • 方法二：运行cmd.exe，输入下方指令，在指定路径生成blog文件夹，即静态网页文件。这里要注意的是，指定的路径必须为空，或者不存在（hugo将自动生成）
hugo new site 路径

  我们来看下生成的blog文件夹的内容：

  这里我们只需要大概了解一下就可以，如果只是使用blog，我们只需要了解页面的基本配置就可以，这个之后也会讲到。

2.主题选择及安装方法：

  Hugo给使用者提供了很多现成的themes，风格各异，侧重也不同，我们只需要安装相应的主题并且配置即可使用，不需要自己设计页面样式，这也是用hugo建站的省力之处。我们可以访问官网以获取主题：Hugo Themes，选择blog分类，挑选你喜欢的主题。我将以我使用的 diary 主题为例介绍下载安装方法：

  • 下载方法一：进入主题页面，点击download，进入github页面，选择code，下载.zip文件，并解压到先前创建的../blog/themes文件夹。接着打开blog\config.toml，添加主题配置，theme="hugo-theme-diary-master”，引号中的命名即themes文件夹中的主题文件夹名。

  • 下载方法二：进入主题页面，点击download，进入github页面，复制网址，打开cmd.exe进入blog/themes文件夹，输入
git clone https://github.com/AmazingRise/hugo-theme-diary

  直接将主题文件下载到thenes文件夹，使用VPN可以解决下载速度慢的问题。

  下载完之后同样在config.toml中修改或添加theme配置，同上，即可使主题生效。

  • 如何判断主题是否生效？ 在完成上述两个方法重的一个后，回到blog目录，输入
hugo server -D

  生成静态文件，并开启本地预览服务器。接着在浏览器中访问http://localhost:1313/，即可预览。显示如下图即说明主题配置成功：
